## How to Contribute

### We welcome contributions in many forms, including:

- Bug reports
- Feature requests
- Code improvements
- Documentation updates

### To contribute, please follow these steps:

1. **Fork the Repository**: Click on the "Fork" button on the top-right of the repository page.
2. **Clone Your Fork**: Clone your forked repository to your local machine.
```sh
git clone https://github.com/your-username/remdep.git
```
3. **Create a Branch**: Create a new branch for your changes.
sh
```sh
git checkout -b feature/your-feature-name
```
4. **Make Changes**: Implement your changes or fixes.
5. **Test Your Changes**: Make sure to test your changes thoroughly. You can run the project locally to verify everything works as expected.
6. **Commit Changes**: Commit your changes with a clear and descriptive message.
```sh
git add .
git commit -m "Add a detailed description of your changes"
```
7. **Push Changes**: Push your changes to your forked repository.
```sh
git push origin feature/your-feature-name
```
8. **Create a Pull Request**: Open a pull request from your forked repository to the original repository. Provide a detailed description of your changes.

## Development Setup
To set up your development environment, follow these steps:

1. **Clone the Repository**:
```sh
git clone https://github.com/your-username/remdep.git
```
2. **Navigate to the Project Directory**:
```sh
cd remdep
```
3. **Install Dependencies**:
```sh
npm install
```
4. **Run the Project: To run the project locally, use the following command**:
```sh
node ./dist/index.js <keywords> [flags]
```

## Coding Standards
### To maintain code quality and consistency, please adhere to the following guidelines:

+ **Use TypeScript**: The project is written in TypeScript. Ensure your code adheres to TypeScript standards.
+ **Follow Existing Style**: Follow the existing code style and conventions used in the project.
+ **Write Clear and Descriptive Commit Messages**: Use clear and concise commit messages that describe the purpose of the changes.
+ **Test Your Code**: Write tests for new features and bug fixes. Ensure all tests pass before submitting a pull request.

## Submitting Changes
### When submitting changes, please ensure:

+ Your code adheres to the coding standards outlined above.
+ You have tested your changes thoroughly.
+ You provide a clear description of your changes in the pull request.
